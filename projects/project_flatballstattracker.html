Introduction

Welcome to the showcase of Grocery Manager. This project is a comprehensive mobile application designed to simplify your shopping experience and eliminate food waste. This app goes beyond basic list-keeping, ensuring you never let another perishable item expire. Enjoy features like real-time chat with certified dietitians for personalized advice, and let the app generate recipes based on your current inventory. Grocery Manager is your all-in-one solution, effortlessly managing your items and offering culinary inspiration to make meal planning a breeze.

Background

To provide context, this three month project is a part of the CPEN 321 curriculum, in which in a group of four, we were tasked with creating a mobile application that met a certain complexity, and followed standard software development procedures. Grocery Manager is a take on addressing the problem of managing perishable food items. It takes the hassle away from deciding what to cook every day, and ensures one stays on top of their items so that knowing expiry dates and when to use what items is never an issue.

Objectives

The main objectives of Grocery Manager are to explore software development procedures, and mobile (Android) development.

Methodology/Approach

In developing Grocery Manager, we employed an agile/waterfall hybrid work approach, utilizing Java in the front-end, and Node.js in the backend to achieve our goals. The development process involved careful planning, prototyping, revision, and constant evaluation.

Features/Functionalities

The key features of Grocery Manager include:

  • Inventory Management, Notifications and Tracking.
  • Recipe Generation.
  • Real Time Chat with a Dietician.

These features are designed to ensure one knows exactly what they have at a time. Furthermore, notifications are sent every time items are about to expire. Inventory habits (frequency, items, etc.) are tracked within an algorithm such that an algorithm can let the user know when it is time to restock on an item.

A set of recipes are generated for the user to have a set of options to choose from when deciding what they want to cook for a meal. This set of recipes are catered towards what is already within the inventory the user provides.

Finally, there is a real time chat feature with a dietician, where a user is able to chat and connect with a dietician for dietary instructions, or advice.

Technologies Used

Grocery Manager utilizes the following technologies:

  • Java (for front-end development)
  • Node.js (for back-end development)
  • Espresso (for front-end testing)
  • Jest (for back-end testing)
  • Firebase (for push notifications)

We also integrated a Recipe Api, a UPC Code Api and Google Sign-In Api to enhance functionality.

Challenges Faced

During the development of Grocery Manager, we encountered several challenges, including time management, group management, and working efficiently on a large scale project. We overcame these challenges by keeping good communication, ensuring we contacted the teaching staff when we needed help, and gained valuable insights in the process.

Results/Outcomes

The results of Grocery Manager include an 80% in the course. This project has allowed me to further explore front-end development, and learn about software development and its lifecycle in the process.

Future Enhancements

Looking ahead, we plan to clean up our codebase, and add additional functionality such as a recommended shopping list. Further plans include: better database management, improvements to the UI/UX, and much more. Your feedback is valuable, and we will consider it for future iterations of the project.

Conclusion

In conclusion, the development of Grocery Manager has been a rewarding journey that allowed us to delve into the intricacies of software development, particularly in the realm of mobile applications. The comprehensive features, including Inventory Management, Recipe Generation, and Real-Time Chat with a Dietician, collectively contribute to a seamless and innovative solution for simplifying the user's shopping experience.

Throughout this project, we embraced a hybrid agile/waterfall methodology, leveraging technologies such as Java and Node.js for front-end and back-end development, respectively. The integration of Firebase for push notifications, a Recipe API, a UPC Code API, and the Google Sign-In API enriched the application's functionality, providing users with a holistic experience.

We encountered and overcame challenges related to time and group management, demonstrating the importance of effective communication and seeking assistance when needed. The project's positive outcome is reflected in the successful completion of the CPEN 321 curriculum, culminating in an 80% achievement in the course.

Looking ahead, we envision refining the codebase, implementing additional features like a recommended shopping list, enhancing database management, and improving UI/UX design. Your feedback is invaluable, and we appreciate the support of collaborators, mentors, and everyone involved in the project.

Thank you for exploring Grocery Manager with us. We are excited about its potential impact and look forward to future iterations that will further elevate the user experience.
